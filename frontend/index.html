하이브리드 영속성

Snapshot + AOF

Snapshot 파일 확인 중 마지막 저장 시각을 불러오는 중입니다. 파일 상태를 불러오는 중입니다.
AOF 파일 확인 중 마지막 기록 시각을 불러오는 중입니다. 파일 상태를 불러오는 중입니다.
복구 흐름 Snapshot -> AOF Replay 시작 시 Snapshot을 먼저 불러오고, 그 뒤 AOF 변경분을 replay 해 최종 상태를 복원합니다. 종료 시 Snapshot 저장 · 비정상 종료 시 AOF가 마지막 변경분을 살립니다.

비정상 종료 복구 데모

AOF로 살아남는 마지막 쓰기

현재 상태 기록 없음 아직 복구 데모 데이터를 쓰지 않았습니다.
마지막으로 쓴 값 - 브라우저에 저장된 기준값
서버에서 읽은 현재 값 - API 재조회 결과

강제 종료 뒤에는 터미널에서 `docker compose up -d api` 를 실행하면 이 화면이 자동으로 복구를 감지합니다.

발표 시연 순서

  1. `복구용 데이터 생성` 으로 메모리와 AOF에 남을 기준값을 만든다.
  2. `API 강제 종료` 를 눌러 graceful shutdown 없이 프로세스를 끊는다.
  3. 프론트 화면은 그대로 둔 채 터미널에서 `docker compose up -d api` 로 API만 다시 올린다.
  4. 이 카드가 자동으로 같은 값을 다시 읽어 `복구 확인` 상태로 바뀌는지 보여준다.
  5. 원하면 아래 Snapshot/AOF 크기와 이벤트 로그까지 함께 보여준다.
